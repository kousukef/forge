# docs/inbox/ -- 未分類知識の一時退避場所

## 目的

分類先が明確でない知識やメモを一時的に退避する場所。ここに配置されたファイルは `/compound` 実行時に自動スキャンされ、`docs/domain/` への移動が提案される。

## 使い方

開発中に得た知見やメモのうち、`docs/domain/` のどのファイルに分類すべきか判断できないものをここに配置する。

- ファイル形式は Markdown（`.md`）を推奨する
- ファイル名は内容がわかる名前をつける（例: `payment-gateway-retry-policy.md`）
- 1ファイル1トピックを原則とする

## /compound による自動スキャン

`/compound` 実行時に以下の処理が自動的に行われる:

1. `docs/inbox/` 内のファイル（README.md を除く）がスキャンされる
2. 各ファイルの内容がドメイン分類テーブルに基づいて分析される
3. 分類可能な知識について `docs/domain/` 配下の適切なファイルへの移動が提案される
4. ユーザーが承認した場合、ファイル内容が移動先に追記され、元ファイルは削除される

分類できないファイルは `docs/inbox/` に残り、次回の `/compound` で再度スキャンされる。Markdown 以外のファイル（画像、PDF 等）はスキャン対象外としてスキップされる。
